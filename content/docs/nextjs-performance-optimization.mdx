## ðŸ”¹ 3. Use Static Site Generation (SSG) When Possible

SSG **pre-renders pages at build time**, making them **blazing fast**.

âœ… **Use `getStaticProps` for static pages:**

```javascript
export async function getStaticProps() {
  const res = await fetch("https://api.example.com/posts");
  const posts = await res.json();

  return { props: { posts }, revalidate: 60 }; // ISR (Incremental Static Regeneration)
}
```

âœ… **When to use SSG?**  
- Content **does not change frequently** (e.g., blogs, landing pages).  
- You need **fast initial load** times.  

---

## ðŸ”¹ 4. Use Server-Side Rendering (SSR) Only When Necessary

SSR generates pages **on each request**, which can slow down performance.

âœ… **Use `getServerSideProps` for dynamic data fetching:**

```javascript
export async function getServerSideProps() {
  const res = await fetch("https://api.example.com/user");
  const user = await res.json();

  return { props: { user } };
}
```

âœ… **When to use SSR?**  
- When **real-time data** is needed (e.g., dashboards, authenticated pages).  

âŒ **Avoid SSR for static pages** to prevent unnecessary re-renders.

## ðŸ”¹ 8. Optimize API Requests

```

```
